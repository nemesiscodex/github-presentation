Better pull request

Pull requests are an excellent tool for fostering code review.

A good practice is for someone else to merge your code into the mainline, ensuring 2 sets of eyeballs review each feature.

This is simple to organise when working in pairs, but in larger teams you may need a system for determining who reviews what.
